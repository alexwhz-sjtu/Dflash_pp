# DFlash++ 迭代式块内生成和草稿树构建

## 1. 目标与范围

本文档给出一套新的 `DFlash++` 设计说明，目标是在现有模型基础上，改进训练目标和推理流程，在一个长度为 `B` 的 draft block 内进行最多 `N` 轮迭代生成，根据置信度构建草稿树/链。


## 2. Baseline：DFlash

1. 大模型是可靠的信息处理器，投机解码中应该尽量减少小模型计算和中间结果因为这并不可靠。大模型跨层融合的 hidden states 已经编码了历史上下文的关键信息，草稿模型可以依赖这些条件作为kvcache而不是自己计算 KV cache。
2. 不同层的 hidden states 关注历史的不同方面，因此多层条件信息的融合仍然是有价值的。
3. 一次性预测整个长度为 `B` 的 草稿block，了和验证对齐，对于越靠前的token，训练时loss越大，计算交叉熵。


## 3. 设计动机

当前一次前向直接预测整个 `B` 长块的方式有一个根本困难：草稿模型必须在没有中间自条件更新的情况下，同时决定块内所有位置的 token。对于越靠后的位置，这种决策既依赖注入的kvcache，又依赖块内前部 token 的具体取值，而后者在单次前向中并未被真正“确认”。

因此，更合理的训练/推理目标不是：

- 一次前向把整个块全部预测正确。

而是：

- 给定注入的kvcache 和一个已确认的块内前缀，可靠地补全剩余后缀；
- 在推理时，通过若干轮块内迭代，把“全未知”逐步转化为“已确认前缀 + 未知后缀”。
- kvcache注入逻辑和dflash保持一致。

1. 一次性预测整个长度为 `B` 的 草稿block，了和验证对齐，对于越靠前的token，训练时loss越大，计算交叉熵。后半段更容易因为语义转折和条件不足而出错；因此更合理的方式是先预测一段可靠前缀，再基于该前缀继续预测剩余部分。现有实验发现，平均来讲，dflash接收长度是小于6的（B=16）时，即，6~16之间大段长度被浪费。如果能在现有基础上多迭代1到2次，而增加2个接收长度，则可以少调用几次目标模型，产生收益。
2. 自适应草稿块解码和草稿树构建。dflash为了和验证对齐，对于越靠前的token，训练时loss越大，计算交叉熵。这就会天然导致，越靠前的token，预测准确率更大而且置信度越大，靠后的token置信度小，这样才能loss最小化。我已通过实验发现，置信度存在这样的规律，且置信度和迭代一次的接收长度强相关。
3. 我计划改进训练方法，实现因果对齐的迭代去噪和草稿树构建。推理时，我的目标是，每次迭代，从最开始的待解码位置开始，通过置信度设计算法，定位潜在拒绝位置（前后取1个token），算法待定。之后如果这个位置已经很靠后了比如大于10，就可以不进行下一轮迭代。如果比较小，则只解码拒绝位置之前的部分，以这三个潜在拒绝点为分支，解码的左侧干净token为前，并行预测块内剩下的位置（即下一次迭代）。这样就有了三个草稿树/草稿链。迭代至多三次。传统草稿树对于每个位置都进行多候选采样，在dflash这种16个长度的超长草稿数量的场景下，2^16个token，数量爆炸。因此我进行了改进。
4. 上面的推理流程就是这样，不过现在不用实现推理，主要关注训练。我想达到这样的效果，我应该怎么训练。我现在的思路是：我的目标是，草稿块内，给定任意长度干净前缀，预测块内剩余token。因此，我准备在dflash模型基础上继续训练，loss分为两部分，一部分是dflash现有的，另一部分为继续性损失L_con。

这使得草稿模型的任务从“一步猜全段”变成“多步扩展连续前缀”。这与当前 speculative decoding 的验收机制天然一致，因为当前 target 侧本来就是按最长连续前缀来接受草稿 token。



## 4. 形式化定义

### 4.1 条件与目标块

记：

- `c` 表示由大模型最新融合 hidden states 得到的条件表示。
- 目标 draft block 为
  $
  x_{1:B} = (x_1, x_2, \ldots, x_B).
  $
- `B` 是单个 draft block 的长度。
- `N` 是块内迭代的最大轮数。
- `[M]` 表示 mask token。

### 4.2 第 `t` 轮块内状态

记第 `t` 轮已经确认的前缀长度为 `p_t`，满足
$
0 \le p_t \le B.
$

则第 `t` 轮输入状态为
$
z^{(t)} = (x_1, \ldots, x_{p_t}, [M], \ldots, [M]).
$

也就是说：

- 前 `p_t` 个位置使用 clean token；
- 后 `B - p_t` 个位置全部为 mask。

草稿模型在条件 `c` 与状态 `z^{(t)}` 下输出块内各位置的分布：
$
q_\theta^{(t)}(\cdot)
=q_\theta(\cdot \mid c, z^{(t)}).
$

对于任意尚未确定的位置 `i > p_t`，模型给出：
$
q_\theta(x_i \mid c, x_{1:p_t}, [M]_{p_t+1:B}).
$

### 4.3 块内迭代更新

在第 `t` 轮，模型先对剩余区间 `p_t+1 : B` 产生预测，再由某个截断准则决定本轮接受多少个新增 token。记该新增长度为 `k_t`，满足
$
0 \le k_t \le B - p_t.
$

更新规则为
$
p_{t+1} = p_t + k_t.
$

若 `p_t = B`，则说明整个 block 已完成；若达到最大轮数 `N` 仍未完成，则由工程策略决定是停止、退化为更保守的接受方式，还是退回最小步长继续生成。

## 5. 训练目标：基于 clean prefix 的 completion 学习

### 5.1 为什么训练前缀必须使用 clean token

这里不引入 prefix 误差鲁棒性训练。理由很直接：

- 在推理时，只有被大模型连续接受的前缀才会成为下一轮的条件；
- 一旦前缀某处出错，target 侧就不会继续接受后续 token；
- 因此，下一轮块内生成真正会条件化到的前缀，应该被视为“已经确认正确的前缀”。

所以训练目标应与实际使用场景严格对齐：前缀部分始终使用 clean token，而不是模型自生成且可能出错的 prefix。

### 5.2 训练样本构造

给定一个目标块 `x_{1:B}`，随机采样一个切分点
$
p \sim \mathcal{P}, \qquad 0 \le p \le B-1.
$

构造训练输入状态：
$
z^{(p)} = (x_1, \ldots, x_p, [M], \ldots, [M]).
$

记剩余长度为
$
R = B - p.
$

模型在条件 `c` 与状态 $`z^{(p)}`$ 下，对剩余位置给出预测：
$q_\theta(\cdot \mid c, z^{(p)})_{p+d}, \qquad d = 1, 2, \ldots, R.$

### 5.3 主损失

定义块内 completion 主损失为
$\mathcal{L}_{\mathrm{pred}}(c, x_{1:B}; p)=
\sum_{d=1}^{R}
\tilde w_d^{(R)}
\cdot
\mathrm{CE}
\Big(
q_\theta(\cdot \mid c, z^{(p)})_{p+d},
x_{p+d}
\Big).
$

整体训练目标为
$\mathcal{L}_{\mathrm{train}}
=\mathbb{E}_{(c, x_{1:B}) \sim \mathcal{D}}
\,\mathbb{E}_{p \sim \mathcal{P}}
\Big[
\mathcal{L}_{\mathrm{pred}}(c, x_{1:B}; p)
\Big].$

其中：

- `CE` 是标准 token-level 交叉熵；
- 每个剩余位置都参与监督；
- 权重 $`\tilde w_d^{(R)}`$ 用于强调“越靠前的未决位置越重要”。

## 6. 归一化的平滑距离衰减权重

### 6.1 指数衰减形式


最推荐的默认形式是指数衰减：
$w_d = \exp\!\left(-\frac{d-1}{\gamma}\right), \qquad d = 1, 2, \ldots, R,$
并做剩余长度归一化：
$\tilde w_d^{(R)}=\frac{w_d}{\sum_{j=1}^{R} w_j}.$

这里 $`\gamma > 0`$ 是衰减温度：

- $`\gamma`$ 越小，前几位的权重越集中；
- $`\gamma`$ 越大，权重越平缓。


### 6.3 为什么需要归一化

若不对 $`w_d`$ 做归一化，不同切分点 `p` 对应的剩余长度 `R` 会直接改变总损失尺度，从而引入额外偏置：

- `R` 较大的样本会天然贡献更大的总 loss；
- `R` 较小的样本会被低估。

归一化后：
$
\sum_{d=1}^{R} \tilde w_d^{(R)} = 1,
$
于是不同剩余长度下的训练样本在总量级上可比，优化器看到的是“同一类 completion 任务的不同难度实例”，而不是“长度越长权重越大”的混合目标。

### 6.4 数学性质

对固定的切分点 `p` 和正权重 $`\tilde w_d^{(R)} > 0`$，上式仍然是一个加权的 proper scoring objective。也就是说，在理想函数类和总体期望意义下，最优解仍然要求每个位置的预测分布匹配真实条件分布；权重改变的是有限模型容量和有限数据下的优化侧重点，而不是 token-level 条件分布的 Bayes 最优形式。

因此，前重后轻权重的作用不是改变正确答案，而是显式告诉模型：

- 如果容量有限，优先把“更早的未决位置”做准；
- 因为这些位置更直接决定最终可接受前缀长度。

## 7. 块内迭代推理算法

### 7.1 推理目标

推理时不再要求一次前向直接生成整个 `B` 长块，而是通过最多 `N` 轮迭代，逐步扩展块内前缀。

设第 `t` 轮已经接受的前缀长度为 `p_t`，则：

1. 输入状态为 $`x_{1:p_t}`$ 加上右侧全 mask；
2. 模型预测剩余区间 `p_t+1 : B`；
3. 截断准则给出本轮新增长度 `k_t`；
4. 将前 `k_t` 个新增 token 视作下一轮的 clean prefix；
5. 直到块完成或达到最大轮数 `N`。

### 7.2 数学描述

记第 `t` 轮模型对剩余位置的点预测为
$
\hat x_{p_t+1:B}^{(t)}.
$

截断准则给出
$
k_t = \mathrm{Cutoff}\!\left(
q_\theta^{(t)},
\hat x_{p_t+1:B}^{(t)},
p_t
\right).
$

然后定义下一轮状态为
$z^{(t+1)}=
\Big(
\hat x_1, \ldots, \hat x_{p_t+k_t},
[M], \ldots, [M]
\Big),
$
其中前缀部分由前几轮累计接受的 token 构成。

### 7.4 工程上的保守回退

若某轮截断准则给出 `k_t = 0`，说明模型认为当前剩余区间整体都不够可靠。此时可以考虑2种保守策略：（和你讨论）

1. 表明难度很难，直接停止该 block 的草稿扩展，让 target 尽快接管；此时是否需要在训练时调整，即模型在挑选中间点时不用挑选最开始的token。
2. 强制退化为 `k_t = 1`，让块内生成最小步长推进；


其中最稳健的通常是第 1 或第 2 种。第 3 种需要谨慎，否则容易把本来不可靠的预测硬塞进前缀。

## 8. 为什么“连续前缀”是唯一合理的块内接受单位

在当前 speculative decoding 验收机制中，大模型接受的是最长连续匹配前缀，而不是若干离散的高置信位置。因此块内迭代时也应只扩展连续前缀。

原因有三点：

1. target 侧的验收本来就是前缀式的。一旦前面某处错了，后面位置即使偶然正确，也不会被继续接受。
2. 若在块内做“散点 unmask”，则后续预测会条件化到一些尚未被 target 连续确认的位置，这与真实可执行推理路径不一致。
3. 连续前缀是唯一和现有验收规则完全同构的块内状态表示。

因此，块内状态不应被定义为“任意集合位置已知”，而应定义为“前 `p` 个位置已知，后面全 mask”。

## 9. 两类截断准则

截断准则的本质是：在没有块内 target 验证的情况下，估计本轮最多能安全扩展多长的连续前缀。

严格地说，理想对象是事件
$A_k^{(t)}=\mathbf{1}
\left[\hat x_{p_t+1:p_t+k}^{(t)}=x_{p_t+1:p_t+k}\right],$
即“本轮新增的前 `k` 个 token 是否全部正确”。

我们真正想估计的是
$
P\!\left(A_k^{(t)} = 1 \mid c, z^{(t)}\right).
$

下面给出两类方案。

### 9.1 方案 A：启发式截断

#### 9.1.1 基本思想

不额外训练新的 head，而是直接利用草稿模型 logits 的统计量构造每个位置的可靠性分数，再据此选出最长连续前缀。

对第 `t` 轮剩余区间中的第 `j` 个新增位置，可定义以下量：

- 最大概率
  $
  p^{\max}_{t,j} = \max_v q_\theta^{(t)}(v)_{p_t+j};
  $
- top-1 与 top-2 的 logit 间隔
  $
  m_{t,j} = \ell^{(1)}_{t,j} - \ell^{(2)}_{t,j};
  $
- 预测熵
  $
  H_{t,j}=\sum_v q_\theta^{(t)}(v)_{p_t+j}
  \log q_\theta^{(t)}(v)_{p_t+j}.
  $

基于这些量，可以构造启发式可靠性分数：
$
r_{t,j}=\beta_1 \log p^{\max}_{t,j}+ \beta_2 m_{t,j}- \beta_3 H_{t,j}- \beta_4 \frac{j-1}{B-p_t},$
其中 `\beta_1, \beta_2, \beta_3, \beta_4 \ge 0` 是可调系数。

若不想引入组合系数，也可以直接使用最简单的 `p^{\max}_{t,j}` 或 `-H_{t,j}` 作为可靠性分数。

#### 9.1.2 最大阈值前缀规则

给定阈值 `\tau_j`，取最大的 `k_t` 满足：
$r_{t,j} \ge \tau_j,\qquad \forall j = 1, 2, \ldots, k_t.
$

也就是：
$
k_t=
\max
\left\{
k :
r_{t,1}, \ldots, r_{t,k}
\text{ 全部通过阈值}
\right\}.
$

这里的阈值可以是：

- 常数阈值 $`\tau_j = \tau`$；
- 位置相关阈值 $`\tau_j = \tau(j)`$；
- 或者用一个常数阈值，再在 $`r_{t,j}`$ 中显式加入位置惩罚项。

#### 9.1.3 风险预算规则

也可以先把每个位置映射成启发式错误风险：
$
e_{t,j} = g(r_{t,j}),
$
其中 `g` 是一个单调递减映射，例如：
$
e_{t,j} = 1 - p^{\max}_{t,j}.
$

然后定义累计风险预算$ `\epsilon`$，取最大的$ `k_t`$ 满足
$
\sum_{j=1}^{k_t} e_{t,j} \le \epsilon.
$

这个规则比逐点阈值更灵活，因为它允许前若干位置中存在轻微波动，只要累计风险仍在预算内。

#### 9.1.4 优缺点

优点：

- 不需要额外训练新的参数；
- 易于实现，适合作为第一版 sanity check；
- 能快速验证“块内迭代生成”本身是否带来收益。

缺点：

- raw confidence 往往未校准；
- 阈值依赖模型规模、温度、采样方式和位置分布；
- 位置之间的相关性没有被显式建模，不能直接等价为 prefix 正确概率。

### 9.2 方案 B：可学习截断

#### 9.2.1 基本思想

引入一个轻量的 acceptance head 或 boundary head，让模型显式预测“本轮最多有多长的新增前缀会整体正确”。

有两种建模方式：

1. token-level 正确率预测；
2. prefix-level 存活概率预测。

其中更推荐第二种，因为它与 speculative decoding 的连续前缀验收机制直接同构。

#### 9.2.2 token-level 方案

对第 `t` 轮剩余区间中第 `j` 个位置，预测：
$
s_{t,j}=
P\!\left(
\hat x_{p_t+j}^{(t)} = x_{p_t+j}
\mid c, z^{(t)}
\right).
$

监督标签可以直接由草稿预测与真值比较得到：
$
y_{t,j}=
\mathbf{1}
\left[
\hat x_{p_t+j}^{(t)} = x_{p_t+j}
\right].
$

训练损失为二元交叉熵：
$
\mathcal{L}_{\mathrm{tok}}=- \frac{1}{R_t}
\sum_{j=1}^{R_t}
\Big(y_{t,j}\log s_{t,j}+ (1-y_{t,j})\log (1-s_{t,j})
\Big),
$
其中 $`R_t = B - p_t`$。

然后可定义一个近似前缀评分，例如
$
\tilde S_{t,k}=
\prod_{j=1}^{k} s_{t,j},
$
再按阈值决定 `k_t`。

这种方法实现简单，但它默认位置间独立，通常不是最理想的前缀建模方式。

#### 9.2.3 prefix-level 方案

更推荐直接建模 prefix 存活概率：
$
S_{t,k}=
P\!\left(
\hat x_{p_t+1:p_t+k}^{(t)}=
x_{p_t+1:p_t+k}
\mid c, z^{(t)}
\right),
\qquad
k = 1, \ldots, R_t.
$

监督标签定义为
$
Y_{t,k}=\mathbf{1}\left[\hat x_{p_t+1:p_t+k}^{(t)}
=x_{p_t+1:p_t+k}
\right].
$

对应损失可写为
$
\mathcal{L}_{\mathrm{prefix}}=- \frac{1}{R_t}
\sum_{k=1}^{R_t}\Big(Y_{t,k}\log S_{t,k}+ (1-Y_{t,k})\log (1-S_{t,k})
\Big).
$

推理时取最大的 `k_t` 满足
$
S_{t,k_t} \ge \tau.
$

#### 9.2.4 单调性约束

prefix 存活概率应满足：
$
S_{t,1} \ge S_{t,2} \ge \cdots \ge S_{t,R_t}.
$

因为“前 `k+1` 个全对”的事件必然蕴含“前 `k` 个全对”。

为保证这一性质，可以使用两种方式：

1. 直接输出 $`S_{t,k}`$，并加一个单调性正则项；
2. 输出条件存活因子 $`h_{t,j} \in (0,1)`$，再定义
   $
   S_{t,k} = \prod_{j=1}^{k} h_{t,j}.
   $

第二种方式天然保证单调性，更适合作为工程实现。
